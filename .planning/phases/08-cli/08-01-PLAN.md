<task type="auto">
  <name>Task 2: Create validate, lint, and compile command handlers</name>
  <files>
    packages/cli/commands/validate.ts
    packages/cli/commands/lint.ts
    packages/cli/commands/compile.ts
  </files>
  <action>
1. Create `packages/cli/commands/validate.ts`:
   - Export `runValidate(filePath: string | undefined, options: { json?: boolean }): number`
   - If no filePath: print usage error via formatError, return 1
   - Resolve path with `resolve(filePath)`, read with `readFileSync(absPath, "utf-8")`
   - Catch read errors: check for ENOENT/EISDIR -> formatError "File not found: {filePath}", return 2. Re-throw unexpected errors.
   - Call `validate(content)` from `@logic-md/core` (pass raw file content string, NOT parsed spec)
   - If result.ok: print formatSuccess("Valid LOGIC.md file"), return 0
   - If not ok: iterate result.errors, print each with file path, line/column if available, message. Return 1.
   - Errors go to stderr (`console.error`), success to stdout (`console.log`)

2. Create `packages/cli/commands/lint.ts`:
   - Export `LintDiagnostic` interface: `{ level: "warning" | "info"; message: string; step?: string }`
   - Export `lintSpec(spec: LogicSpec): LintDiagnostic[]` as a pure function for testability
   - Lint checks to implement:
     a. Steps without descriptions -> level: "info"
     b. Unreachable steps via `resolve(spec.steps)` -> level: "warning" (only if dagResult has errors with unreachable type... actually check: the DAG resolve returns `{ ok: false, errors }` where errors have a `type` field. Check the DagError type to see if "unreachable" is a valid type. If not, check if unreachable steps appear in a different field of DagResult. Adapt accordingly based on the actual core API.)
     c. Branches without a default -> level: "warning" (check `step.branches?.length && !step.branches.some(b => b.default)`)
     d. Missing fallback section on spec -> level: "info"
     e. Steps with `needs` referencing non-existent steps -> level: "warning" (DAG resolver already catches this, but lint should also flag it)
   - Export `runLint(filePath: string | undefined, options: { json?: boolean }): number`
   - Read file (same ENOENT handling as validate, exit 2)
   - Call `parse(content)` to get LogicSpec. If parse fails, print errors, return 1.
   - Call `lintSpec(spec)` on the parsed result
   - Print diagnostics using formatWarning/formatInfo based on level
   - If `--json` flag: output `JSON.stringify(diagnostics, null, 2)` to stdout instead
   - Return 0 if no warnings (info-only is fine), return 1 if any warnings exist

3. Create `packages/cli/commands/compile.ts`:
   - Export `runCompile(filePath: string | undefined, options: { json?: boolean }): number`
   - Read file (same ENOENT handling, exit 2)
   - Pipeline: parse(content) -> validate(content) -> resolveImports(spec, dirname(absPath)) -> resolve(spec.steps) if steps exist
   - If any step fails: print errors via formatError, return 1
   - On success: output the fully-resolved LogicSpec as JSON to stdout (`JSON.stringify(spec, null, 2)`)
   - If DAG levels are available, include them in output as a `_dagLevels` property (or print as a separate info line)
   - Data output goes to stdout so it can be piped. Status messages go to stderr.
   - IMPORTANT: `resolveImports(spec, basedir)` needs `dirname(resolve(filePath))` as basedir, not CWD
  </action>
