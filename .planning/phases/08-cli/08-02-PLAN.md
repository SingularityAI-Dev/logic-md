  <action>
1. Replace the existing `packages/cli/cli.test.ts` (which only tests VERSION re-export) with comprehensive CLI integration tests.

2. Build the CLI first in a `beforeAll`: run `npx tsc -p tsconfig.json` from packages/cli directory so dist/cli.js exists.

3. Create a helper function `run(args: string[], env?: Record<string, string>)` that:
   - Uses `execFileSync("node", [cliPath, ...args], { encoding: "utf-8", env: { ...process.env, ...env } })`
   - Wraps in try/catch to capture both stdout and stderr from the error object
   - Returns `{ stdout, stderr, exitCode }`
   - The CLI path should be `resolve(__dirname, "dist/cli.js")` or equivalent

4. Test groups (describe blocks):

   a. **"validate command"**:
      - "exits 0 for valid file": run validate on fixtures/valid.logic.md, assert exitCode 0, stdout contains success message
      - "exits 1 for invalid file": run validate on fixtures/invalid.logic.md, assert exitCode 1, stderr contains "error"
      - "exits 2 for missing file": run validate on "nonexistent.logic.md", assert exitCode 2, stderr contains "not found"
      - "exits 1 with no file argument": run validate with no args, assert exitCode 1

   b. **"lint command"**:
      - "exits 0 for clean file": run lint on fixtures/valid.logic.md (designed to have zero warnings), assert exitCode 0
      - "reports warnings for lint issues": Create an inline fixture or a third fixture file that has lint issues (step without description, missing fallback, branches without default). Assert exitCode 1, stderr contains "warning"
      - "supports --json flag": run lint with --json on a file with issues, assert stdout is valid JSON array

   c. **"compile command"**:
      - "exits 0 and outputs JSON for valid file": run compile on fixtures/valid.logic.md, assert exitCode 0, stdout is valid JSON (JSON.parse doesn't throw)
      - "exits 1 for invalid file": run compile on fixtures/invalid.logic.md, assert exitCode 1
      - "exits 2 for missing file": run compile on "nonexistent.logic.md", assert exitCode 2

   d. **"global flags"**:
      - "--help prints usage": run with --help, assert exitCode 0, stdout contains "Usage" or "logic-md"
      - "--version prints version": run with --version, assert exitCode 0, stdout contains version string
      - "unknown command exits 1": run with "foobar" command, assert exitCode 1

   e. **"NO_COLOR support"**:
      - "output has no ANSI codes when NO_COLOR is set": run validate on invalid file with env NO_COLOR=1, assert stderr does NOT contain "\x1b["
      - "output has ANSI codes when NO_COLOR is unset": run validate on invalid file without NO_COLOR, assert stderr contains "\x1b[" (skip if CI disables colors)

5. Keep the existing VERSION re-export test in a separate `index.test.ts` (don't delete it -- it lives in a different file).
